## ğŸ§  Testing Approach

### 1. Manual Testing
Manual testing is performed first to:
- Understand application behavior
- Identify functional defects
- Design effective automation scripts later

Artifacts created:
- Test Plan
- Test Scenarios
- Test Cases
- Bug Reports
- Test Summary Report

---

### 2. Test Design Techniques Used
- Equivalence Partitioning
- Boundary Value Analysis (BVA)
- State Transition Testing
- Decision Table Testing

---

### 3. Automation Testing
Automation is used for:
- Regression testing
- Repeated execution of stable test cases

Implemented using:
- Java
- Selenium WebDriver
- TestNG
- Page Object Model (OOPs concepts)

---

### 4. Database Testing
Database testing is performed using **PostgreSQL** to:
- Validate data integrity
- Verify CRUD operations
- Ensure UI actions reflect correctly in the database

---

### 5. Compatibility Testing (Key Focus)
This project includes compatibility testing across:

- Operating Systems: Windows XP (conceptual), Windows 7, Windows 8, Windows 11
- Browsers: Chrome, Firefox, Edge

Compatibility testing ensures:
- Backward compatibility
- Identification of OS-specific or browser-specific defects
- Real-world enterprise readiness

---

### 6. CI/CD Integration
Automation tests are integrated with **Jenkins** to:
- Run tests automatically after code changes
- Detect failures early
- Support continuous testing
